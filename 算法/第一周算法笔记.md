## 🚫 一、Java 语法避坑避错清单（重点整理）

在手写代码时，很容易因为语法不熟练写出编译 Bug。请在平板上重点标注这三个最常用对象的 API 差异：

### 1. 长度/大小获取方式（大厂常考低级语法点）
| 对象类型 | 获取长度的语法 | 示例 | 备注 |
| :--- | :--- | :--- | :--- |
| **基本数组** (如 `int[]`) | `.length` | `nums.length` | **属性**，千万不要加括号 `()` |
| **字符串** (`String`) | `.length()` | `s.length()` | **方法**，必须带括号 `()` |
| **集合类** (`List`, `Map`, `Set`) | `.size()` | `list.size()` / `map.size()` | **方法**，必须带括号 `()` |

### 2. 元素访问与操作方法
*   **数组**：直接用中括号 `nums[i]` 访问和赋值。
*   **字符串**：**不支持**中括号！必须使用 `s.charAt(i)` 读取字符，且字符串是**不可变的**，不能直接通过下标修改。
*   **ArrayList**：**不支持**中括号！使用 `list.get(i)` 读取，`list.add(element)` 添加，`list.size()` 获取长度。
*   **HashMap**：使用 `map.put(key, value)` 存入，`map.get(key)` 读取，`map.containsKey(key)` 判断键是否存在。

### 3. "=" 与 "==" 的天坑
*   `=`：**赋值**（将右边的值给左边，如 `left = temp + 1;`）。
*   `==`：**比较**（判断两边是否相等，返回 boolean，如 `if (x == y)`）。

## 💡 二、五道核心算法题精提炼

### 1. 两数之和 (Two Sum)

*   **核心概念**：**哈希表单次遍历（One-pass HashMap）**。
*   **为什么快？(Why)**：
    *   普通双循环时间复杂度为 $O(N^2)$。
    *   哈希表解法通过“**边遍历边放入**”的滚雪球方式，将查找互补数（`complement = target - nums[i]`）的时间降为 $O(1)$，总时间复杂度优化为 $O(N)$。
*   **去重与防自匹配**：边遍历边查询，在当前元素还没放入 Map 时就去查它的互补数，天然避免了“同一个元素使用两次”的 Bug。
*   **Java 核心模板**：
    ```java
    Map<Integer, Integer> map = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (map.containsKey(complement)) {
            return new int[] { map.get(complement), i };
        }
        map.put(nums[i], i);
    }
    ```

### 2. 删除有序数组中的重复项 (Remove Duplicates)

*   **核心概念**：**快慢指针（Slow/Fast Pointers）**。
*   **原地修改限制**：题目强制要求空间复杂度为 $O(1)$，意味着**禁止使用任何额外的 `ArrayList`**。
*   **双指针分工**：
    *   `slow`（慢指针）：代表“**新领地**”的边界。只有遇到新数字时，新领地才扩张（`slow++`）。
    *   `fast`（快指针）：作为“**侦察兵**”，一刻不停向右扫描，遇到新数字就送回给 `slow`（`nums[slow] = nums[fast]`）。
*   **Java 核心模板**：
    ```java
    int slow = 0;
    for (int fast = 1; fast < nums.length; fast++) {
        if (nums[fast] != nums[slow]) {
            slow++;
            nums[slow] = nums[fast];
        }
    }
    return slow + 1; // 唯一元素的个数是 slow 下标 + 1
    ```
*   **为什么去重后数组后面有脏数据？**：Java 中数组长度固定。我们无法缩减物理数组的大小，只能把去重后的 `k` 个元素挪到最前面，后面的元素保持原样，面试官和判题系统只会检查前 `k` 个元素。

### 3. 三数之和 (Three Sum)

*   **核心概念**：**排序 + 双指针左右夹逼**。
*   **为什么哈希表不适用？**：三数之和的核心难点在**“去重”**。如果用哈希表找匹配，最后为了给结果去重，必须对每一个三元组排序并做哈希，时间开销极大，容易超时。
*   **排序的妙用**：通过 `Arrays.sort(nums)` 先排序，将相同的数聚在一起，去重就变得极度简单。
*   **指针夹逼逻辑**：
    *   固定第一个数 `nums[i]`（若 `nums[i] > 0` 直接中断退出，因为三正数相加不可能为 0）。
    *   **去重 `i`**：`if (i > 0 && nums[i] == nums[i-1]) continue;`。
    *   设定 `left = i + 1`, `right = nums.length - 1`，计算 `sum = nums[i] + nums[left] + nums[right]`。
    *   `sum < 0` ➡️ `left++`；`sum > 0` ➡️ `right--`。
    *   `sum == 0` ➡️ 记录答案，并执行 **`left` 和 `right` 指针的去重**（跳过相邻相同元素），最后双向收缩。
*   **Java 核心模板**：
    ```java
    Arrays.sort(nums);
    for (int i = 0; i < nums.length - 2; i++) {
        if (nums[i] > 0) break;
        if (i > 0 && nums[i] == nums[i - 1]) continue;
        int left = i + 1, right = nums.length - 1;
        while (left < right) {
            int sum = nums[i] + nums[left] + nums[right];
            if (sum == 0) {
                ans.add(Arrays.asList(nums[i], nums[left], nums[right]));
                while (left < right && nums[left] == nums[left + 1]) left++; // 去重 left
                while (left < right && nums[right] == nums[right - 1]) right--; // 去重 right
                left++; right--;
            } else if (sum < 0) left++;
            else right--;
        }
    }
    ```

### 4. 长度最小的子数组 (Minimum Size Subarray Sum)

*   **核心概念**：**滑动窗口（双指针区间控制）**。
*   **`ArrayList.remove(0)` 的致命瓶颈**：
    *   用 `ArrayList` 物理存储窗口数据，剔除元素时调用 `remove(0)` 会导致整个数组在内存中**向左平移**，单次平移时间复杂度为 $O(W)$，导致整体算法退化到 $O(N^2)$ 发生超时。
*   **虚拟窗口优化**：使用 `left` 和 `right` 指针，只用一个 `sum` 累加器记录区间和。
    *   **右侧进水**：`right` 不断右移，`sum += nums[right]`。
    *   **左侧排水（核心逻辑）**：当 `sum >= target` 时，开始收缩左侧：记录当前窗口长度，`sum -= nums[left]`，然后 `left++`，循环直至 `sum < target`。
*   **Java 核心模板**：
    ```java
    int left = 0, sum = 0, minLength = Integer.MAX_VALUE;
    for (int right = 0; right < nums.length; right++) {
        sum += nums[right];
        while (sum >= target) {
            minLength = Math.min(minLength, right - left + 1);
            sum -= nums[left];
            left++;
        }
    }
    return minLength == Integer.MAX_VALUE ? 0 : minLength;
    ```

### 5. 无重复字符的最长子串 (Longest Substring Without Repeating)

*   **核心概念**：**滑动窗口 + 位置映射哈希表**。
*   **抛弃内层扫描循环**：如果用循环从 `left` 到 `right - 1` 找重复，时间复杂度为 $O(N^2)$。我们使用 `Map<Character, Integer> lastSeen` 记录字符最后一次出现的下标，实现 $O(1)$ 的瞬间查找。
*   **`Math.max` 防回退陷阱（高频考点）**：
    当字符 `c` 触发重复时，我们想让 `left` 跳到 `lastSeen.get(c) + 1`。但是，**如果这个重复字符的上次出现位置已经在当前 `left` 左侧（窗口外面）**，直接跳转会导致 `left` 往回倒退，把脏数据重新吸纳进来。因此必须使用：
    `left = Math.max(left, lastSeen.get(c) + 1);`
*   **Java 核心模板**：
    ```java
    int left = 0, maxLength = 0;
    Map<Character, Integer> lastSeen = new HashMap<>();
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (lastSeen.containsKey(c)) {
            left = Math.max(left, lastSeen.get(c) + 1); // 核心：防回退
        }
        lastSeen.put(c, right);
        maxLength = Math.max(maxLength, right - left + 1);
    }
    return maxLength;
    ```

                                   │
         ┌─────────────────────────┴─────────────────────────┐
         ▼                                                   ▼
     双指针夹逼 (Two Pointers)                             滑动窗口 (Sliding Window)

  │ 适用: 寻找有序数组中匹配的对 │                     │ 适用: 寻找连续子数组/子字符串 │
